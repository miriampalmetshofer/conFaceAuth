# conFaceAuth - Continuous Face Authentication

This repository contains a basic prototype for continuous face authentication pipeline. 

## Overview

The system provides two main execution modes:
1. **Live Mode** - For quick testing and real-time authentication using your webcam
2. **Batch Processing Mode** - For processing multiple pre-recorded participants videos 

## Quick Start Guide

Make sure to execute commands from the `src` folder.

### Prerequisites
- Python 3.7+
- Required packages (install via `pip install -r requirements.txt`)

### Quick Testing with live.py

For quick testing and trying out the system, using `live.py` is recommended. This script will:
- Access your webcam
- Perform real-time face authentication
- Display the authentication results on screen

#### Setting Up Enrollment Data for Quick Testing

Before running live.py, you should create a folder with some images of yourself taken by your desktop camera to use as enrollment data:

1. Create a folder to store your enrollment images (e.g., `data/my_enrollment`)
2. Take several photos of yourself using your webcam and save them in this folder
3. Update the `enrollment_folder` path in `src/live_config.json`:

```json
"enrollment": {
  "enrollment_folder": "../data/my_enrollment"
}
```

### Batch Processing with main.py

For comprehensive evaluation, use `main.py`. 
CHeck the `bulk_config.json` file to set up the batch processing parameters.

## Configuration

The system uses two configuration files:

1. `config.json` - Used by live.py for quick testing
2. `bulk_config.json` - Used by main.py for batch processing

## Project Structure

- `src/` - Source code directory
  - `face_auth/` - Core authentication components
  - `analysis/` - Jupyter notebooks to analyse results.csv generated by the batch processing
  - `live.py` - Script for live webcam authentication
  - `main.py` - Script for batch processing
  - `live_config.json` - Configuration for live mode
  - `bulk_config.json` - Configuration for batch processing
- `data/` - Directory for storing data
  - Create your enrollment folder here
